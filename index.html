Custom STL Containers

Rebuilding the C++ Standard Library – From the Ground Up ✨

🚀 Project Overview

This project is not just a clone—it's a reimagination of core C++ STL containers.
Designed for education, performance insight, and deeper understanding, it covers custom-built vector, map, set, stack, and more— all built using low-level C++ with templates, OOP, and Red-Black Trees.

💡 Key Highlights & Technical Superiority

  • 🌱 Custom Vector: Built from raw pointers with manual memory handling, dynamic resizing, and iterator support.
  • 🧠 Manual Memory Management: Understand how capacity grows and memory is freed—unlike STL which abstracts this away.
  • ⚙️ Red-Black Tree: Fully implemented with insertions, deletions, rotations, and balancing without using built-in trees.
  • 🔑 Custom Map & Set: Both containers built using the Red-Black Tree to support associative structures with sorted order.
  • 📦 Stack & Pair: Lightweight and efficient implementations mimicking STL behavior with custom logic.
  • 🧩 STL vs Custom: No allocator, no prebuilt headers. Every function, every structure, crafted by hand.
  • 🧵 Template Programming: Generic containers that work with all types, showing advanced C++ techniques.
  • 🎯 Debug-Oriented: Rich comments, modular file structure, and visual logs make this perfect for learning.

🔍 Differences from Standard STL

  • Transparency: Unlike STL, where implementation details are hidden, this project exposes every working mechanism.
  • No Allocators: This project doesn’t use STL allocators, giving you full control over memory behavior.
  • Educational Perspective: STL is for usage—this project is for learning how STL is designed from scratch.
  • Code Ownership: Understand every line. Debug every flow. Own your container logic completely.
